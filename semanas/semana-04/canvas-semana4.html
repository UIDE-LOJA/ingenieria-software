Esta semana introducimos DevOps y Metodologías Ágiles con foco en Scrum y Kanban, gestión de backlog y flujo de trabajo con Git. Veremos la cultura DevOps, diferencias entre desarrollo plan-driven vs ágil, prácticas de grooming y planning, y el uso básico de Git para trabajo colaborativo en equipos de desarrollo.

¿Qué veremos?

Tema Contenido clave Ejemplos
Cultura DevOps DevOps: integración de desarrollo y operaciones. Principios: colaboración, automatización, medición, compartir (CAMS). Beneficios: entregas más rápidas, mayor calidad, mejor comunicación. Ciclo de vida: Plan → Code → Build → Test → Release → Deploy → Operate → Monitor. CI/CD pipelines; Infrastructure as Code; monitoreo continuo; feedback loops; equipos cross-funcionales.
Plan-driven vs Ágil Plan-driven: requisitos fijos, planificación detallada upfront, cambios costosos. Ágil: requisitos evolutivos, planificación iterativa, adaptación al cambio. Cuándo usar cada uno: proyectos críticos vs innovación, equipos grandes vs pequeños. Plan-driven: sistemas bancarios, aeroespacial. Ágil: startups, productos digitales, aplicaciones web/móvil.
Scrum y Kanban Scrum: sprints time-boxed (2-4 semanas), roles (PO, SM, Dev Team), ceremonias (planning, daily, review, retro). Kanban: flujo continuo, WIP limits, visualización, pull system. Scrumban: híbrido de ambos. Scrum: sprints de 2 semanas, daily standup 15min. Kanban: tablero con columnas (To Do, In Progress, Done), WIP limit 3.
Backlog y Planning Product Backlog: lista priorizada de features. Grooming: refinamiento de historias, estimación, clarificación. Sprint Planning: selección de historias, definición de sprint goal, descomposición en tareas. DoR/DoD: Definition of Ready y Done. User stories con criterios INVEST; story points (Fibonacci); planning poker; DoR: criterios claros, estimada; DoD: code review, tests pasando.
Git Básico Conceptos: repositorio, commit, branch, merge, pull request. Flujo: clone → branch → commit → push → PR → review → merge. Buenas prácticas: commits atómicos, mensajes descriptivos, ramas cortas, code review. git clone, git checkout -b feature/login, git commit -m "feat: add login", git push, crear PR en GitHub, code review, merge to main.

Objetivos de aprendizaje

Al finalizar la semana podrás:

  • Describir la cultura DevOps y su importancia en el ciclo de desarrollo de software moderno.

  • Comparar Scrum y Kanban y aplicar sus principios en la gestión de proyectos ágiles.

  • Implementar prácticas básicas de Git para trabajo colaborativo en equipos de desarrollo.

  • Analizar críticamente cómo escalar metodologías ágiles en diferentes contextos organizacionales.
