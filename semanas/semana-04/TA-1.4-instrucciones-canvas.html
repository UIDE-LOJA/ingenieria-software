Esta actividad de trabajo autónomo te permitirá planificar y organizar un proyecto de software mediante la elaboración de un roadmap y tablero con épicas. Te enfocarás en definir fases del proyecto, crear épicas con historias de usuario, y establecer criterios DoR/DoD para el equipo de desarrollo.

¿Qué realizarás?

Componente Instrucciones específicas Criterios de calidad
Información del proyecto Selecciona un proyecto de software (puede ser el proyecto de la materia). Incluye nombre, descripción breve y objetivos principales. Proyecto bien definido y relevante.
Roadmap del proyecto Crea roadmap visual con 4 fases: Inception, Elaboration, Construction, Transition. Define hitos y timeline (semanas/meses). Roadmap claro con timeline realista.
Épicas del proyecto Define mínimo 3 épicas principales del proyecto. Cada épica debe tener: título, descripción, valor de negocio, y estimación (story points). Épicas bien definidas y priorizadas.
Historias de usuario Para cada épica, descompón en 3-5 historias de usuario siguiendo formato: "Como [rol] quiero [acción] para [beneficio]". Incluye criterios de aceptación. Historias siguiendo criterios INVEST.
Definition of Ready (DoR) Define checklist de DoR: criterios que una historia debe cumplir antes de entrar al sprint (ej: criterios de aceptación claros, estimada, sin dependencias bloqueantes). DoR específico y aplicable.
Definition of Done (DoD) Define checklist de DoD: criterios para considerar una historia terminada (ej: código revisado, tests pasando, documentación actualizada, desplegado en staging). DoD completo y medible.

Estructura del roadmap:

  • Fase 1 - Inception (Semanas 1-2): Definición de visión, stakeholders, requerimientos iniciales
  • Fase 2 - Elaboration (Semanas 3-5): Arquitectura, prototipos, validación técnica
  • Fase 3 - Construction (Semanas 6-12): Desarrollo iterativo, sprints, entregas incrementales
  • Fase 4 - Transition (Semanas 13-14): Testing final, despliegue, capacitación

Ejemplo de Épica:

Épica: Gestión de Usuarios
Descripción: Sistema completo de autenticación y autorización de usuarios
Valor de negocio: Alto - Funcionalidad core del sistema
Estimación: 21 story points

Historias de usuario:
1. Como usuario nuevo quiero registrarme para acceder al sistema (5 pts)
2. Como usuario registrado quiero iniciar sesión para usar la aplicación (3 pts)
3. Como usuario quiero recuperar mi contraseña para acceder si la olvido (3 pts)
4. Como administrador quiero gestionar roles para controlar permisos (5 pts)
5. Como usuario quiero actualizar mi perfil para mantener mi información (5 pts)

Ejemplo de DoR y DoD:

Definition of Ready:

  • Historia escrita en formato estándar (Como... quiero... para...)
  • Criterios de aceptación definidos y claros
  • Historia estimada por el equipo
  • Sin dependencias bloqueantes
  • Mockups o wireframes disponibles (si aplica)

Definition of Done:

  • Código implementado y funcional
  • Tests unitarios escritos y pasando
  • Code review completado y aprobado
  • Documentación técnica actualizada
  • Desplegado en ambiente de staging
  • Criterios de aceptación verificados

Materiales: Presentación Semana 4, material sobre Scrum/Kanban, ejemplos de roadmaps. Herramientas: Miro, Lucidchart, Figma, o herramientas de diagramación.
