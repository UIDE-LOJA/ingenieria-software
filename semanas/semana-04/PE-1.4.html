Esta actividad práctica te permitirá aplicar los conceptos de DevOps y metodologías ágiles mediante la simulación de Scrum/Kanban y el uso básico de Git. Te enfocarás en gestionar un backlog, crear issues y pull requests, y trabajar colaborativamente usando herramientas modernas de desarrollo.

¿Qué realizarás?

Actividad Instrucciones específicas Entregable
Preparación individual Revisa presentación Semana 4 sobre Scrum, Kanban, y flujo de trabajo con Git. Crea cuenta en GitHub si no la tienes. Cuenta GitHub activa y comprensión de conceptos.
Configuración del tablero En grupos de 3-4, crea un tablero Kanban en GitHub Projects con columnas: Backlog, To Do, In Progress, Review, Done. Tablero Kanban configurado.
Creación de issues Crea 5 issues (historias de usuario) con título, descripción, criterios de aceptación y etiquetas (feature, bug, enhancement). 5 issues bien documentados.
Simulación Sprint Asigna issues a miembros del equipo, mueve tarjetas por el tablero simulando un sprint de 30 minutos. Practica daily standup. Tablero con movimiento de tarjetas.
Flujo Git básico Cada miembro: crea rama, hace commit, crea Pull Request vinculado a un issue, y otro miembro hace code review. Mínimo 2 PRs creados y revisados.
Entrega final Captura pantallas del tablero, issues, PRs y breve reflexión (máximo 200 palabras) sobre la experiencia con Scrum/Kanban y Git. PDF con evidencias y reflexión.

Ejemplo de estructura de Issue:

Título: [US-001] Como usuario quiero registrarme en el sistema

Descripción:
Como usuario nuevo
Quiero poder registrarme en el sistema
Para acceder a las funcionalidades de la aplicación

Criterios de Aceptación:
- [ ] Formulario con campos: nombre, email, contraseña
- [ ] Validación de email único
- [ ] Contraseña mínimo 8 caracteres
- [ ] Mensaje de confirmación al registrarse
- [ ] Redirección a página de login

Etiquetas: feature, priority-high
Estimación: 3 story points

Comandos Git básicos:

# Crear y cambiar a nueva rama
git checkout -b feature/nombre-funcionalidad

# Hacer cambios y commit
git add .
git commit -m "feat: descripción del cambio"

# Subir rama al repositorio
git push origin feature/nombre-funcionalidad

# Crear Pull Request desde GitHub
# Vincular PR con issue usando: "Closes #numero-issue"
