Esta actividad te permitirá escribir historias de usuario siguiendo el formato estándar y aplicando los criterios INVEST. Incluirás criterios de aceptación claros para cada historia.

¿Qué realizarás?

Componente Instrucciones específicas Criterios de calidad
Historias de usuario Escribe 5 historias de usuario para tu proyecto usando el formato: "Como [rol], quiero [funcionalidad], para [beneficio]". 5 historias con formato correcto.
Criterios de aceptación Para cada historia, define 3-5 criterios de aceptación usando formato "Dado que... Cuando... Entonces..." (Gherkin). 3-5 criterios por historia.
Validación INVEST Verifica que cada historia cumpla con INVEST (Independent, Negotiable, Valuable, Estimable, Small, Testable). Marca con ✓ cada criterio. Tabla INVEST completa.

Formato de historia de usuario:

Como [rol de usuario]
Quiero [funcionalidad/acción]
Para [beneficio/valor]

Ejemplo de historia de usuario:

Historia #1: Registro de usuario

Como usuario nuevo
Quiero registrarme en la plataforma con mi correo electrónico
Para acceder a las funcionalidades del sistema

Criterios de aceptación:

  1. Dado que estoy en la página de registro, cuando ingreso un correo válido y contraseña, entonces el sistema crea mi cuenta.
  2. Dado que ingreso un correo ya registrado, cuando intento registrarme, entonces el sistema muestra error "Correo ya existe".
  3. Dado que ingreso una contraseña débil, cuando intento registrarme, entonces el sistema solicita contraseña más segura.

Criterios INVEST:

  • Independent (Independiente): La historia puede desarrollarse sin depender de otras.
  • Negotiable (Negociable): Los detalles pueden ajustarse con el equipo.
  • Valuable (Valiosa): Aporta valor al usuario o negocio.
  • Estimable (Estimable): Se puede estimar el esfuerzo de desarrollo.
  • Small (Pequeña): Puede completarse en un sprint.
  • Testable (Verificable): Se puede probar si funciona correctamente.

Especificaciones técnicas de entrega

Aspecto Especificación requerida
Formato de archivo PDF con 5 historias de usuario, criterios de aceptación y tabla INVEST. Nomenclatura: TA-1.3_ApellidoNombre_HistoriasUsuario.pdf
Contenido 5 historias de usuario con formato estándar, 3-5 criterios de aceptación por historia (formato Gherkin), tabla INVEST completa.
Extensión 2-3 páginas máximo.

Información importante

  • Modalidad: Trabajo autónomo individual — 2 pts.
  • Extensión: 2-3 páginas.
  • Entrega: Plataforma institucional antes de la fecha límite especificada.
  • Penalización por retraso: -0.25 puntos por cada día de retraso.
