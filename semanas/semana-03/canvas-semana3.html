Esta semana abordamos Metodologías Ágiles con enfoque en historias de usuario, casos de uso y criterios de aceptación. Exploraremos técnicas de validación de software, métodos de trazabilidad y administración de requerimientos, fundamentales para el desarrollo de software de calidad.

¿Qué veremos?

Tema Contenido clave Ejemplos
Historias de Usuario Técnica ágil de Programación Extrema (XP) para expresar requerimientos en tarjetas. Formato: "Como [usuario] quiero [acción] para [beneficio]". Criterio INVEST: Independent, Negotiable, Valuable, Estimable, Small, Testable. Historia: "Como productor agrícola quiero publicar mis productos para llegar a más compradores". Priorización por cliente, descomposición en tareas.
Casos de Uso Técnica formal UML introducida en el método Objectory (Jacobson et al., 1993). Identifica actores, interacciones y detalles del sistema. Diagramas visuales con actores y casos de uso. Sistema Marketplace Agrícola: Actores (Productor, Comprador, Administrador) con casos de uso (Publicar productos, Buscar productos, Gestionar transacciones).
Criterios de Aceptación Forman parte del contrato del sistema entre cliente y desarrollador. Definen qué hace aceptable un sistema. En XP, el cliente define pruebas y el desarrollo no avanza hasta pasarlas. Criterios medibles y verificables: "El sistema debe integrarse con Azure DevOps sin desarrollos adicionales". Base para pruebas de aceptación.
Métodos de Validación Validación vs Verificación (Boehm, 1979): "¿Construimos el producto correcto?" vs "¿Construimos bien el producto?". Técnicas: revisiones e inspecciones, prototipos, pruebas de aceptación. Revisiones estáticas de código, prototipos iterativos para validar conceptos, pruebas de aceptación con datos reales del cliente.
Trazabilidad y Administración Trazabilidad: registrar relaciones entre requerimientos, fuentes y diseño. Administración de requerimientos: proceso de comprender y controlar cambios. Identificación única, proceso de cambio, políticas de seguimiento. Matriz de trazabilidad: REQ-001 → CP-REQ01-PUB → Implementación. Gestión de impacto de cambios, documentación continua.

Objetivos de aprendizaje

Al finalizar la semana podrás:

  • Comprender los fundamentos de requerimientos del sistema y técnicas basadas en escenarios.

  • Dominar las historias de usuario en metodologías ágiles aplicando el criterio INVEST.

  • Analizar los casos de uso como técnica UML formal para modelar interacciones del sistema.

  • Definir criterios de aceptación efectivos que formen parte del contrato del sistema.

  • Aplicar métodos de validación (revisiones, prototipos, pruebas) y gestionar la trazabilidad de requerimientos.

Conceptos clave para recordar

  • Validación vs Verificación: "¿Construimos el producto correcto?" vs "¿Construimos bien el producto?" (Boehm, 1979).
  • Criterio INVEST: Independent, Negotiable, Valuable, Estimable, Small, Testable.
  • Trazabilidad: Registrar relaciones entre requerimientos, fuentes y diseño del sistema.
  • Ciclo de vida de requerimientos: Elicitación → Especificación → Validación → Gestión (proceso iterativo).
